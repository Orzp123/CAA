# Feature Specification: 多租户用户认证与权限管理系统

**Feature ID**: 001-multi-auth-tenant-rbac
**Created**: 2026-06-17
**Status**: Draft

---

## Overview

为 CAA 平台构建统一的用户认证与权限管理体系，支持多种登录方式（账号密码、微信、第三方单点登录），通过统一的认证网关对所有请求进行身份验证和权限校验。租户以学校为单位隔离，系统设有总 ADMIN 租户进行全平台治理。不同账户类型（系统管理员、学校管理员、教师、学生）拥有不同权限，权限可通过管理页面配置，且仅限总 ADMIN 修改。

---

## Problem Statement

平台需要服务于多所学校的教师和学生，各学校的认证方式各不相同（微信、校园 SSO、账号密码），账户权限因角色和学校而异。现有系统缺乏统一的身份认证入口、租户隔离机制和细粒度的权限管理能力，导致安全风险和运维成本高。

---

## Goals

- 提供统一认证网关，所有服务的请求均经过身份验证和权限校验后才能访问
- 支持三种登录方式，各租户可独立配置默认登录方式和允许使用的登录方式
- 以学校为租户单位隔离数据和权限，总 ADMIN 租户拥有全平台治理能力
- 支持四种账户类型，权限可通过管理界面配置，变更仅限总 ADMIN 操作
- 登录安全性满足验证码防爆破要求
- JWT 签名算法可通过配置选择，无需重新部署

---

## Non-Goals

- 不包含账户的自注册功能（账户由管理员创建或通过 SSO/微信首次登录自动创建）
- 不包含多因素认证（MFA）
- 不包含账户注销或 GDPR 数据删除流程
- 不包含审计日志功能（后续迭代）

---

## User Scenarios & Testing

### 场景 1：学生使用账号密码登录

1. 学生访问平台登录页
2. 选择「账号密码」登录方式
3. 输入学号、所属学校、密码及验证码
4. 系统验证通过后跳转到学生工作台
5. **验收**：账号密码登录耗时 ≤ 3 秒（可配），错误密码提示友好，连续失败达阈值后账户锁定

### 场景 2：微信用户首次登录

1. 用户点击「微信登录」按钮
2. 跳转微信授权，完成授权后回到平台
3. 系统检测为首次登录，自动在「微信大学」租户下创建账户，补全必填信息（学号、姓名等）
4. 完成资料填写后进入平台
5. **验收**：新账户自动归属微信大学租户，账户类型可选，必填字段校验完整

### 场景 3：第三方 SSO 用户登录

1. 用户点击对应学校的「SSO 登录」入口
2. 跳转至该学校 SSO 系统完成认证
3. 回调后系统在对应学校租户下查找或创建账户
4. 用户进入所属学校的工作台
5. **验收**：不同学校 SSO 回调能正确映射到对应租户

### 场景 4：总 ADMIN 配置租户权限

1. 总 ADMIN 登录后访问「权限管理」页面
2. 选择某个学校租户，配置该租户可访问的功能模块
3. 保存后，该租户下的所有用户立即生效新权限
4. **验收**：普通学校 ADMIN 无法访问权限管理页面，操作被拒绝并提示无权限

### 场景 5：学校 ADMIN 配置本校账户权限

1. 学校 ADMIN 登录后访问本校用户管理页面
2. 为教师角色添加「课程管理」权限
3. 保存后该校教师立即获得对应权限
4. **验收**：学校 ADMIN 仅能看到本校用户和本校权限范围内的配置项

### 场景 6：切换登录方式

1. 某学校租户配置默认登录方式为「账号密码」，同时允许「微信登录」
2. 用户访问该学校登录页，默认显示账号密码表单
3. 用户点击「微信登录」切换到微信授权流程
4. **验收**：不支持的登录方式入口不显示；切换后登录流程正常

## Functional Requirements

### 认证网关

- FR-01：所有后端服务的 API 请求必须经过统一认证网关，网关负责 Token 验证和权限校验；验证通过后原样透传 Token，下游服务自行解析 Token 获取用户信息；Token 校验失败时返回 401（未认证：无 Token 或 Token 无效/过期），权限不足时返回 403（已认证但无权限访问该资源）
- FR-02：网关放行白名单路径（登录接口、验证码接口、公开资源），白名单通过配置中心维护，变更后热加载生效，无需重启
- FR-03：Token 签名算法可通过配置中心选择，支持 Spring Security 内置的对称和非对称算法；变更后热加载生效，新生成的 Token 使用新算法，无需重启服务
- FR-04：Token 采用固定过期时间，过期后需重新登录，过期时长通过配置中心调整，热加载生效，不支持自动续签；Token 过期后前端显示"登录已过期，请重新登录"提示，3 秒后自动跳转登录页
- FR-04b：系统提供单设备登录限制配置，分为全局配置、租户级配置、账户类型级配置三层；租户未配置时继承全局配置，账户类型未配置时继承租户配置；启用时，新设备登录后该账号旧设备的 Token 立即失效，旧设备收到专属提示"您的账号已在其他设备登录，当前会话已失效"后跳转登录页

### 登录方式

- FR-05：支持账号密码登录，凭据为「学号 + 学校 + 密码」组合；密码长度为 8～20 个字符，必须同时包含数字和字母
- FR-06：支持微信 OAuth2 登录，登录用户自动归属「微信大学」租户
- FR-07：支持第三方单点登录（SSO），SSO 用户归属其对应学校租户
- FR-08：登录页面支持在多种登录方式间切换，切换逻辑由该租户的可用登录方式配置决定
- FR-09：每个租户可独立配置：默认登录方式（单选）和可用登录方式列表（多选）；系统强制要求每个租户至少配置一种可用登录方式，保存时校验，不允许空配置

### 验证码

- FR-10：账号密码登录时必须输入图形验证码；微信 OAuth2 和第三方 SSO 登录无需验证码，依赖外部系统自身的安全机制；用户可点击验证码图片刷新获取新验证码
- FR-11：连续登录失败达到阈值后，该账户进入锁定状态，锁定期间拒绝所有登录请求；连续失败次数阈值和锁定时长均通过配置中心调整，热加载生效（默认：失败 5 次，锁定 15 分钟），超时后自动恢复启用
- FR-12：验证码有效期为 5 分钟，一次性使用

### 账户模型

- FR-13：账户必填字段：平台唯一 ID（UUID 格式，随机生成，全平台唯一，无业务语义）、姓名、昵称（未填时默认等于姓名）、所属学校、账户类型、学号
- FR-13b：账户状态分为三种：启用（正常）、禁用（管理员主动停用，不可登录）、锁定（系统因连续登录失败自动触发，超时后自动解锁）；账户被禁用后，该账户所有在线 Token 立即失效，网关拒绝其后续请求
- FR-14：学号在同一学校租户内唯一
- FR-15：账户类型分为四种：系统 ADMIN、学校 ADMIN、教师、学生
- FR-16：微信/SSO 首次登录时若账户不存在，系统自动创建账户并要求补全必填字段；若用户中途放弃补全，账户数据不保存，下次登录重新走首次创建流程；微信登录创建的账户默认类型为「学生」；SSO 登录时系统尝试从 IdP 返回的属性中识别账户类型，识别失败则默认为「学生」

### 租户模型

- FR-17：系统存在一个全局 ADMIN 租户，其下系统 ADMIN 账户拥有全平台操作权限；系统 ADMIN 权限硬编码内置，不经过权限配置页面，不可被修改或缩减
- FR-17b：学校租户在系统部署时通过初始化数据预置；总 ADMIN 可通过管理页面后续新增、修改、停用学校租户；租户被停用后，其下所有账户立即无法登录，已颁发的 Token 自然过期后失效，数据保留，可恢复；网关设计为无状态，支持多实例水平扩展（本期单实例部署，Phase 2 实现多实例）
- FR-18：每个学校对应一个租户，学校租户下的学校 ADMIN 仅能管理本校范围内的数据和配置
- FR-19：「微信大学」为微信登录用户专属租户
- FR-20：不同租户可独立配置其可访问的功能模块权限

### 权限管理

- FR-21：权限分为两层：租户级权限（租户可访问哪些功能模块）和角色级权限（账户类型在已开放模块内可执行哪些操作）。计算规则为租户优先：租户未开放的功能模块，无论角色权限如何均直接拒绝；租户已开放的模块内，由角色级权限决定具体操作许可
- FR-22：租户权限和角色权限均可通过管理页面可视化配置
- FR-23：权限的新增、修改、删除操作仅限总 ADMIN 账户执行
- FR-24：学校 ADMIN 无权访问权限管理页面，尝试访问时提示无权限并拒绝
- FR-25：权限变更在用户重新登录后生效；Token 有效期内权限以颁发时快照为准
- FR-28：功能模块是权限分配的基本单元，系统部署时内置一套默认功能模块；总 ADMIN 可通过管理页面在默认值基础上新增、修改、停用功能模块

### 租户识别

- FR-27：系统支持两种租户识别方式：1）多域名模式——每个学校租户绑定独立子域名，访问时自动识别租户；2）主域名选择模式——用户访问主域名时通过学校选择器选择所属学校；两种方式可同时启用，子域名访问优先级高于主域名选择器

### 错误与降级处理

- FR-26：微信 OAuth2 或第三方 SSO 服务不可用时，系统显示"当前登录方式暂不可用，请尝试其他方式"提示，登录页其余可用登录方式入口保持正常，不阻断整个登录页

---

## Success Criteria

- 账号密码登录全流程耗时不超过 3 秒（不含网络传输时间）
- OAuth2/SSO 外部跳转回调后的平台处理耗时不超过 5 秒
- 权限变更后，用户重新登录后感知到权限更新
- 连续登录失败锁定机制：100% 的暴力破解尝试在第 5 次失败后被阻断
- 不同学校租户的用户数据严格隔离，跨租户数据访问请求 100% 被拒绝
- Token 算法配置项变更后，无需重启服务即可对新生成的 Token 生效
- 管理员完成权限配置操作的页面交互不超过 3 步

## Key Entities

| 实体 | 关键属性 |
|------|---------|
| 租户 (Tenant) | id、名称、类型（ADMIN/学校/微信）、默认登录方式、可用登录方式列表 |
| 账户 (Account) | 平台唯一 id、学号、姓名、昵称、所属租户、账户类型、密码哈希、状态 |
| 权限 (Permission) | id、名称、资源标识、操作类型 |
| 角色权限配置 | 租户 id、账户类型、权限列表 |
| 租户功能配置 | 租户 id、可访问功能模块列表 |
| Token 配置 | 签名算法、过期时间 |

## Clarifications

### Session 2026-06-17

- Q: 第三方 SSO 平台支持哪些协议（SAML 2.0、OAuth2/OIDC 或同时支持）？ → A: 仅支持 OAuth2/OIDC，SAML 2.0 暂不实现，后续按需扩展
- Q: Token 刷新策略行为（单 Token 滑动窗口 / 双 Token / 单 Token 固定过期）？ → A: 单 Token 固定过期，过期后需重新登录，无自动续签；新增可配置同一账号是否限制同一时间仅一台设备在线
- Q: 租户级权限与角色级权限的有效值计算规则（交集/并集/租户优先）？ → A: 租户优先——租户未开放的功能模块直接拒绝访问，租户已开放的模块内完全依赖角色级权限控制操作
- Q: 账户状态模型及流转规则？ → A: 三态模型（启用/禁用/锁定），锁定阈值（连续失败次数）和锁定时长均可通过配置项调整
- Q: 外部 OAuth/SSO 服务不可用时的降级行为？ → A: 提示"当前登录方式暂不可用，请尝试其他方式"，引导切换可用登录方式，不阻断整个登录页

## Assumptions

- 第三方 SSO 协议本期仅支持 OAuth2/OIDC，SAML 2.0 暂不实现，后续按需扩展
- 微信登录仅指微信 OAuth2 网页授权，不包含微信小程序
- 「学校」信息由系统预置，不支持用户自行注册新学校
- 权限粒度以「功能模块 + 操作类型」为最小单元，不做字段级权限控制
- 账户密码存储使用行业标准哈希方案，不在规格层面指定具体算法

---

## Dependencies

- 统一认证网关需在现有后端服务之前部署，所有服务需适配网关的 Token 传递方式
- 微信登录需要微信开放平台的 AppID 和 AppSecret 配置
- 第三方 SSO 需要各学校提供 IdP 元数据或回调配置
- 管理界面依赖前端权限配置页面开发

---

## Out of Scope

- 账户自注册
- 多因素认证（MFA）
- 账户数据导出 / GDPR 合规流程
- 登录行为审计日志
- 账户头像和扩展资料管理
